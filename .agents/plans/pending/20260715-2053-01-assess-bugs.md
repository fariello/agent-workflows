## Findings (severity + evidence; verified against source)

No Blocker/High defects found. The install path is no-clobber, backs up before any overwrite, and
stages-not-commits, so the writes themselves are safe; the gaps are in exit-code honesty, rollback
completeness, and batch isolation. Confirmed-SAFE areas (do NOT touch): `is_filename_safe` (no bypass),
the payload-blind `parse_envelope_header`, the `ACK_WRITER` totality/disjointness, `validate_ack`,
`plans.py` Set/Order parsing+sort+warnings, `versioning.py` rc/dev ordering + `next_version_ok`,
`config._preserve_home`, `discovery` ignore-before-recurse, `pypi_links`, `term.should_color`, and the
`run_checks` honesty invariant.

- F4 [MEDIUM] `engine.run()` discards its computed exit code. It sets `returncode = 1` on
  `ensure_repo_root` failure and on a git-diagnostics abort, but the multi-repo loop ends with a
  hardcoded `return 0` (engine.py:~2836). So `install-workflows.py --repo good bad` (bad = not a
  directory, or a diagnostics abort) reports SUCCESS (exit 0) despite a skipped/failed repo. The CLI
  paths propagate `returncode` correctly; only this engine entry point drops it - an entry-point parity
  defect. FIX: `return returncode`. TEST: `engine.run(ns)` with a nonexistent repo in `repo_roots` ->
  assert return 1.
- F5 [MEDIUM] Incomplete rollback: `.created-files.json` omits the `create_setup_artifacts` files.
  `newly_created` (engine.py:~2699) is built only from `installed` items ending ` [install]`; the files
  from `create_setup_artifacts` (`.gitleaksignore`, `.github/workflows/secret-scan.yml`,
  `.agents/comms/.gitignore`, `.agents/comms/README.md`, all `shared/**/.gitkeep`) are returned in
  `artifacts` and never recorded, so `install-workflows.py --undo` leaves them on disk while reporting
  "Rollback completed successfully." (This surface grew this session: the D81 comms skeleton is among the
  orphaned files.) FIX: fold the `artifacts` entries (strip ` [..]`/dry-run suffixes) into `newly_created`
  before `save_created_files_record`. TEST: install into a temp repo, assert `.gitleaksignore` +
  `.agents/comms/README.md` exist, run rollback, assert they are gone.
- F8 [MEDIUM] Batch-isolation gap: `_install_all` (cli.py:~494) and the `setup` loop (cli.py:~748) use
  `except Exception`, but `install_into_repo` -> `install_all` can raise `SystemExit` (directory-conflict
  at engine.py:~924, or a `git_run` failure), which is `BaseException`, NOT `Exception`. So a `SystemExit`
  from one repo escapes the per-repo loop and aborts the whole batch - breaking the R-4 "one repo failing
  must not stop the batch" guarantee. `_run_install` already catches `SystemExit` explicitly; the other
  two paths do not. FIX: in `_install_all` and the `setup` loop, catch `SystemExit` too (e.g.
  `except (Exception, SystemExit)`), matching `_run_install`. TEST: monkeypatch `install_into_repo` to
  raise `SystemExit` for the first of two repos; assert the second still processes and it reports 1 failed.
- F1 [LOW] `prompt_and_run_commit(..., artifacts: list[str] = None)` (engine.py:1884) - annotation
  `list[str]` but default `None`; should be `list[str] | None = None`. Runtime-safe (callers always pass a
  list; body guards `if artifacts:`), but it is the LSP diagnostic deferred all session and the ONLY
  instance of this class in engine.py/cli.py (lane 1 audited the rest). FIX: `list[str] | None = None`.
  TEST: an `inspect.signature`-based guard test asserting no non-Optional param defaults to None (optional).
- F6 [LOW] Preserved-customization mis-tagged. When a user declines to overwrite a manually-customized
  shim, engine.py:~805 appends ` [already current]`, which `format_output_item` maps to `[no change]` -
  so a deliberately-preserved differing file is reported identically to an untouched one. FIX: use a
  distinct tag (e.g. ` [preserved: manual edits]`) + a mapping in `format_output_item`. TEST: customized
  shim + non-interactive `yes=False` -> skipped entry tagged preserved, not "already current".
- F14 [LOW] `save_created_files_record` swallows an `OSError` write failure silently (engine.py:~1700),
  degrading `--undo` fidelity with no signal (compounds F5). FIX: log a stderr warning on failure. TEST:
  simulate an unwritable path -> a warning is emitted.
- F-tools [LOW] Portability/hardening in the workflow tools (batch): (a) CSV outputs opened without
  `newline=""` (`scan_secrets.py:~828`, `run_checks.py:~679`, `bench_env.py:~682`) -> double blank rows on
  Windows; add `newline=""`. (b) `run_checks.py:~239` dead PHONY-skip condition
  (`... and target.startswith(".")` is always False for the reachable names) -> drop the dead clause so
  `all`/`default` aggregate targets are actually skipped. (c) `scan_secrets.redact()` reveals up to 8 of N
  chars for 9-12-char secrets (head4+tail4) -> for `len < 16`, mask more aggressively so the "never a full
  secret" promise holds for short tokens. Each with a small regression test.
