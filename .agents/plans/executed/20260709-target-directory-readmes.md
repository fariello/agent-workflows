# IPD: Standardize developer-facing READMEs in installed framework directories

- Date: 2026-07-09
- Concern: Usability, documentation clarity, and developer safety in target repositories.
- Scope: Define standard READMEs for four directories (`.agents/workflows/`, `.opencode/commands/`, `.claude/commands/`, `workflow-artifacts/`) and update the installer engine to manage them correctly.
- Status: EXECUTED 2026-07-09

## Goal

Provide clear, developer-facing guidance within the target repository directories created or expected by `agent-workflows`. This prevents developer confusion, warns against manual editing of auto-generated shims, and explains the purpose of the `workflow-artifacts/` folder.

## Why

1. **Safety in generated shim folders**: Developers looking at `.opencode/commands/` or `.claude/commands/` might try to manually edit the markdown shims. We need to warn them that these folders are managed and overwritten by the installer.
2. **Git status clarity for `workflow-artifacts/`**: New developers or team members might see a new `workflow-artifacts/` directory and want to `.gitignore` it. We must explain that run records are committed deliverables representing the review history.
3. **Framework orientation**: Anyone browsing the `.agents/workflows/` directory should quickly understand that it holds neutral agent instructions, and know how to trigger or update them.

## Proposed Design

We will introduce README files for the four key directories. All templates will be bundled in the source distribution under `.agents/workflows/templates/` so that the installer code remains clean and free of inline markdown strings. The installer engine (`engine.py`) will handle each directory differently based on its lifecycle:

### 1. Workflow Definitions Directory (`.agents/workflows/README.md`)
* **Behavior**: Treat this as a standard framework asset.
* **Installer Logic**: The file will exist in the source `.agents/workflows/` directory of the `agent-workflows` distribution and will be copied over naturally by the existing installer file sync logic (just like `index.md` or `VERSION`).

### 2. Slash-Command Shim Directories (`.opencode/commands/README.md` and `.claude/commands/README.md`)
* **Behavior**: Sourced from a template and updated/written dynamically by the installer.
* **Installer Logic**: 
  * The installer will read the template from `<source_root>/templates/shim-README.md` and write it to `.opencode/commands/README.md` and `.claude/commands/README.md` during the shim generation phase (`generate_shim_members`).
  * The pruning logic in `prune_stale` will be updated to explicitly exclude `README.md` from deletion, treating it as a known framework shim file.

### 3. Workflow Run Artifacts Directory (`workflow-artifacts/README.md`)
* **Behavior**: Create-if-missing (non-destructive).
* **Installer Logic**:
  * If a `README.md` **already exists** in `workflow-artifacts/`, the installer must **not** overwrite or modify it.
  * If it does not exist, the installer will read the template from `<source_root>/templates/workflow-artifacts-README.md` and write it to `workflow-artifacts/README.md`.
  * Ensure this file is staged with git (`git add`) if it is newly created.

## Directory README Content Specs (Source Templates)

### `.agents/workflows/templates/shim-README.md`
```markdown
# Generated Slash Commands

This directory contains auto-generated slash-command shims for developer agents.

⚠️ **DO NOT EDIT THESE FILES MANUALLY.**
Any modifications will be overwritten or pruned the next time the agent-workflows installer is run.

To install, update, or prune commands, run:
  aw install <dir>
```

### `.agents/workflows/templates/workflow-artifacts-README.md`
```markdown
# Workflow Run Artifacts

This directory contains review records, verification evidence, and audit logs generated by AI agent workflows (such as release reviews or plan reviews).

## ⚠️ Git Guidelines
* **DO NOT gitignore this folder.**
* These records represent the review and approval history of this repository and are intended to be committed to git alongside your code changes.
```

## Required Validation

### Automated Tests
* Add test cases in `tests/test_installer.py` asserting:
  * That a fresh install successfully creates `README.md` files in `.agents/workflows/`, `.opencode/commands/`, `.claude/commands/`, and `workflow-artifacts/`.
  * That `workflow-artifacts/README.md` is NOT modified/overwritten when running the installer if the file already exists and has custom content.
  * That `README.md` files in `.opencode/commands/` and `.claude/commands/` are NOT deleted/pruned by `prune_stale()`.

### Manual Validation
1. **Fresh Install**: Running `aw install` creates all four README files, and stages them with git.
2. **Re-run (Idempotent)**: Running the installer again does not modify `workflow-artifacts/README.md` even if the user has customized it, but regenerates the shims' README files.
3. **Prune Check**: Files inside `.opencode/commands/` that are not shims are pruned, but `README.md` remains intact.
4. **Dry Run**: Running with `--dry-run` reports the creation/update of these files without writing to disk.

## Open Questions (Resolved)

1. **Should we use block markers in `workflow-artifacts/README.md`?**
   * *Resolution*: Keep it simple. A "write once if missing" rule is less intrusive and gives the repo owner complete control over the file.
2. **Do we bundle the shims/artifacts README files in the source distribution, or write them inline?**
   * *Resolution*: Bundled in the source. Templates will live in the source tree under `.agents/workflows/templates/` and be packaged with the distribution.
